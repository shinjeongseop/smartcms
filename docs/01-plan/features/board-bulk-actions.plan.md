# 게시판 다중 선택 삭제/복사/이동 기능 추가 계획

> Feature: board-bulk-actions
> Phase: Plan
> Target: smartcms PHP 게시판

## 1. 목적

게시판 목록에서 여러 게시글을 선택한 뒤 한 번에 삭제, 복사, 이동할 수 있게 한다.
운영자가 반복 작업을 줄이고, 기존 단건 삭제 권한과 게시판 접근 권한을 유지한 상태로 안전하게 게시글을 관리하는 것이 목표다.

## 2. 현재 구조 요약

- 게시판 목록 컨트롤러: `board/index.php`
- 게시글 보기 및 단건 삭제 처리: `board/view/index.php`
- 게시판 공통 함수: `common/board.php`
- 기본 게시판 목록 스킨: `skins/board/default/list.php`
- 유튜브 게시판 목록 스킨: `skins/board/youtube/list.php`
- 단건 삭제 함수: `smartcms_board_delete_post()`
- 게시글 관리 권한 판단: `smartcms_board_can_manage_post()`, `board_manage_level`
- 게시판 목록은 스킨 메타의 `layout` 값에 따라 table, cards, webzine 형태로 출력된다.

## 3. 범위

### 포함

- 게시판 목록에서 게시글 다중 선택 UI 추가
- 선택한 게시글 일괄 삭제
- 선택한 게시글을 대상 게시판으로 복사
- 선택한 게시글을 대상 게시판으로 이동
- 작업 전 CSRF 검증
- 작업별 권한 검증
- 대상 게시판 존재 여부 및 쓰기/관리 권한 검증
- 첨부파일과 에디터 이미지 복사/이동 정책 정의 및 구현
- 작업 결과 메시지 표시
- 감사 로그 기록
- 기본 스킨과 유튜브 스킨 목록 화면 반영

### 제외

- 댓글만 별도 복사/이동하는 기능
- 검색 결과 전체 화면에서 게시판을 가로지르는 일괄 작업
- 휴지통/복구 UI
- 비동기 일괄 작업 큐
- 관리자 전용 별도 일괄 관리 화면
- 게시판 설정 화면 개편

## 4. 기능 요구사항

### 다중 선택

- 게시판 목록에서 관리 권한이 있는 사용자에게만 선택 체크박스를 노출한다.
- 전체 선택 체크박스를 제공한다.
- 선택된 게시글이 없으면 작업을 실행하지 않고 안내 메시지를 표시한다.
- 공지글도 선택 가능하되 기존 게시글 관리 권한을 그대로 따른다.

### 일괄 삭제

- 각 게시글마다 `smartcms_board_can_manage_post()` 기준으로 삭제 가능 여부를 확인한다.
- 삭제 가능한 게시글만 처리하고, 실패 건수와 사유를 결과 메시지에 포함한다.
- 기존 `smartcms_board_delete_post()`를 재사용해 댓글, 첨부파일, 에디터 이미지 정리 흐름을 유지한다.

### 일괄 복사

- 대상 게시판을 선택해야 한다.
- 대상 게시판은 활성 상태이고 사용자가 작성 또는 관리 권한을 가져야 한다.
- 원본 게시글의 제목, 내용, 링크, 공지/비밀 여부, 작성자 정보, 작성일 정책을 명확히 정한다.
- 첨부파일은 실제 파일을 새 경로로 복사하고 `board_files` 레코드를 새 게시글 기준으로 생성한다.
- 댓글은 1차 범위에서 복사하지 않는다.
- 복사된 게시글은 대상 게시판의 게시글로 조회 가능해야 한다.

### 일괄 이동

- 대상 게시판을 선택해야 한다.
- 원본 게시판과 대상 게시판이 같으면 처리하지 않고 안내한다.
- 원본 게시글 관리 권한과 대상 게시판 작성 또는 관리 권한을 모두 확인한다.
- 게시글, 첨부파일, 댓글의 `board_id`를 대상 게시판 기준으로 갱신한다.
- 업로드 경로가 게시판별로 분리되어 있으면 실제 파일 이동 또는 기존 경로 유지 정책을 설계 단계에서 확정한다.

## 5. 비기능 요구사항

- 기존 단건 삭제 동작을 변경하지 않는다.
- PHP 파일 안에 새 `<script>` 구현을 넣지 않고 필요한 이벤트는 별도 JS 파일로 분리한다.
- 인라인 스타일을 추가하지 않는다.
- 공통 스타일이 필요하면 `common/css/common.css`를 우선 검토한다.
- 없는 DOM 요소 접근에는 null 방어를 둔다.
- `btn-outline-*` 버튼을 사용하지 않는다.
- 대량 처리 중 일부 실패가 전체 화면 오류로 이어지지 않도록 건별 결과를 누적한다.
- DB 작업은 가능한 한 트랜잭션으로 묶고, 파일 작업 실패 시 결과를 명확히 남긴다.

## 6. 구현 대상 초안

- `board/index.php`
  - POST 요청에서 `bulk_delete`, `bulk_copy`, `bulk_move` 액션 처리
  - CSRF 검증, 권한 검증, 대상 게시판 검증, flash 메시지 설정

- `common/board.php`
  - 선택 게시글 ID 정규화 함수
  - 일괄 삭제 함수
  - 게시글 복사 함수
  - 게시글 이동 함수
  - 첨부파일 복사/이동 보조 함수
  - 일괄 작업 감사 로그 함수 또는 기존 감사 로그 재사용

- `skins/board/default/list.php`
  - 선택 체크박스, 전체 선택, 작업 선택, 대상 게시판 선택, 실행 버튼 추가
  - table, cards, webzine 레이아웃 모두에서 선택 UI가 깨지지 않도록 반영

- `skins/board/youtube/list.php`
  - 유튜브 스킨 목록에도 동일한 선택/일괄 작업 UI 반영

- `common/js/board-bulk-actions.js`
  - 전체 선택, 선택 수 표시, 작업별 대상 게시판 활성화, 제출 전 확인 처리

## 7. 데이터 모델

신규 테이블은 만들지 않는다.

기존 테이블 사용:

- `boards`
- `board_posts`
- `board_comments`
- `board_files`
- 감사 로그 저장 테이블이 있으면 기존 로그 테이블 재사용

복사 시 고려 필드:

- `board_id`
- `author_id`
- `author_name`
- `title`
- `link_url_1`
- `link_url_2`
- `content`
- `content_mode`
- `is_notice`
- `is_secret`
- `attachment_count`
- `created_at`
- `updated_at`

## 8. 권한 정책

- 목록 접근 권한이 없는 게시판에서는 일괄 작업 UI와 처리를 제공하지 않는다.
- 삭제와 이동의 원본 게시글 권한은 `smartcms_board_can_manage_post()`를 따른다.
- 복사와 이동 대상 게시판은 `board_write_level` 또는 `board_manage_level` 기준을 설계 단계에서 확정한다.
- 관리자 권한 사용자는 모든 게시글에 대해 일괄 작업 가능해야 한다.
- 일반 작성자는 본인 글에 대해서만 삭제/복사/이동 가능하게 할지, 관리 권한 전용으로 제한할지 설계 단계에서 확정한다.

## 9. UX 요구사항

- 목록 상단 또는 하단에 일괄 작업 도구 영역을 둔다.
- 선택 수를 표시한다.
- 삭제는 확인 절차를 둔다.
- 복사/이동은 대상 게시판을 선택해야 실행 가능하다.
- 처리 후 기존 게시판 목록으로 돌아오고 결과 메시지를 보여준다.
- 모바일 화면에서도 체크박스와 작업 버튼이 겹치지 않아야 한다.

## 10. 성공 기준

- 관리 권한 사용자가 여러 게시글을 선택해 삭제할 수 있다.
- 관리 권한 사용자가 여러 게시글을 다른 게시판으로 복사할 수 있다.
- 관리 권한 사용자가 여러 게시글을 다른 게시판으로 이동할 수 있다.
- 권한이 없는 사용자는 UI가 보이지 않거나 서버에서 거부된다.
- CSRF 토큰이 없거나 유효하지 않으면 작업이 거부된다.
- 복사된 게시글의 첨부파일 다운로드가 정상 동작한다.
- 이동된 게시글의 보기, 댓글, 첨부파일 다운로드가 정상 동작한다.
- 일부 게시글 실패 시 성공/실패 건수가 명확히 표시된다.
- 기존 단건 삭제, 글쓰기, 수정, 댓글 작성 기능이 유지된다.

## 11. 리스크 및 대응

- 파일 복사 실패: DB insert 전에 파일 복사 가능 여부를 확인하고 실패 건은 건너뛴다.
- DB와 파일 상태 불일치: 트랜잭션과 사후 정리 로직을 설계 단계에서 명확히 한다.
- 스킨별 UI 중복: 공통 partial 또는 함수화 가능성을 먼저 검토한다.
- 대량 처리 성능: 1차 구현은 현재 페이지 선택분 처리로 제한한다.
- 권한 정책 모호성: 설계 단계에서 관리자 전용인지 작성자 포함인지 확정한다.
- 첨부파일 경로 정책 차이: 기존 업로드 디렉터리 구조를 확인한 뒤 복사/이동 정책을 확정한다.

## 12. 검증 계획

- 게시글 0개 선택 후 실행
- 게시글 1개 선택 후 삭제
- 게시글 여러 개 선택 후 삭제
- 첨부파일 포함 게시글 복사
- 첨부파일 포함 게시글 이동
- 비밀글과 공지글 복사/이동/삭제
- 권한 없는 사용자 POST 직접 요청
- CSRF 누락 요청
- 원본 게시판과 동일한 대상 게시판 이동 요청
- table, card, webzine, youtube 목록 레이아웃 확인
