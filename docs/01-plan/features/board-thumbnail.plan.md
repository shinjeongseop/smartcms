# 게시판 썸네일 재설계 PDCA Plan

## 1. 개요

게시판 썸네일 생성과 본문 이미지 출력 정책을 다시 정리한다.
현재는 썸네일 생성 자체보다도, 화면 표시 단계에서 고정 비율 박스와 `object-fit: cover`가 결합되면서 세로가 긴 이미지가 잘리거나, 에디터 업로드 이미지가 원본보다 낮은 품질로 저장되는 문제가 함께 발생하고 있다.

## 2. 문제 정의

- 리스트 썸네일은 잘 보이지만, 본문 이미지와 동일한 정책으로 처리되면 세로가 긴 이미지가 잘릴 수 있다.
- 본문 노출은 크롭이 아니라 원본 비율 유지가 우선인데, 현재는 화면 단계에서 잘림이 발생할 수 있다.
- 에디터 업로드 이미지가 저장 시점에 리사이즈되면 원본 품질이 저하될 수 있다.
- 썸네일 캐시 정책이 단일해서, 리스트/본문/최신글처럼 서로 다른 요구를 분리하기 어렵다.

## 3. 현재 관찰

- `common/image.php`의 리사이즈 함수는 원칙적으로 비율 유지 축소 방식이다.
- `board/editor-upload/index.php`는 업로드된 이미지를 원본 파일에 즉시 리사이즈 저장한다.
- `skins/board/default/list.php`, `skins/board/default/view.php`, `skins/board/youtube/view.php`는 썸네일 이미지를 `ratio` 박스와 `object-fit-cover`로 감싸고 있다.
- `common/board.php`의 본문 렌더링은 에디터 HTML을 그대로 살리지만, 업로드 정책과 화면 표시 방식이 분리되어 있지 않다.

## 4. 목표

- 리스트용 썸네일과 본문용 이미지 정책을 분리한다.
- 세로가 긴 이미지도 본문에서는 잘리지 않게 한다.
- 에디터 업로드 원본 품질을 보존한다.
- 기존 썸네일 캐시와 게시판 스킨 구조를 크게 흔들지 않는다.
- 유창화 썸네일 계열의 GD 기반 리사이즈/캐시 구조는 재사용하되, 사용 지점을 분리한다.

## 5. 범위

### 포함

- 썸네일 생성 방식 재검토
- 리스트/웹진/갤러리/유튜브 스킨의 썸네일 표시 정책 정리
- 본문 이미지 출력 정책 정리
- 에디터 업로드 이미지 저장 정책 정리
- 캐시 정리 기준 검토

### 제외

- 게시판 CRUD 전면 개편
- 관리자 권한 체계 재설계
- 이미지 저장소 구조 완전 교체

## 6. 검토할 대안

### 대안 1. 리스트 전용 크롭, 본문은 원본 유지

- 리스트/카드/갤러리만 썸네일을 사용한다.
- 본문은 원본 또는 폭 맞춤 표시만 한다.
- 가장 단순하고 안전하다.
- 기존 구조를 덜 건드린다.

### 대안 2. 크롭형과 비크롭형 썸네일을 분리

- 리스트용은 `cover` 성격의 썸네일을 유지한다.
- 본문/최신글/에디터 미리보기는 `contain` 성격의 썸네일을 별도로 둔다.
- 화면별 표현을 세밀하게 제어할 수 있다.
- 캐시 종류가 늘어나지만 품질 문제가 가장 적다.

### 대안 3. 업로드 원본 보존 + 렌더 단계에서만 크기 제어

- 에디터 업로드 이미지는 원본 그대로 저장한다.
- 리스트 썸네일만 별도 생성한다.
- 본문과 에디터 이미지는 CSS와 HTML 속성으로만 반응형 처리한다.
- 품질은 가장 좋지만 저장 용량과 트래픽이 늘 수 있다.

## 7. 우선 검토 방향

- 에디터 업로드 원본은 더 이상 저장 시점에 강제 축소하지 않는 방향이 우선이다.
- 본문 이미지는 크롭하지 않고, 이미지 자체의 비율을 유지한다.
- 리스트 썸네일은 화면 균형을 위해 별도 정책을 유지하되, 세로형 이미지가 중요한 스킨에서는 `contain` 또는 세로 여백 허용 방식을 검토한다.
- 유창화 썸네일의 GD 리사이즈 기반은 유지하되, 용도를 분리하는 방식으로 재활용한다.

## 8. 성공 기준

- 본문에서 세로가 긴 이미지가 잘리지 않는다.
- 리스트 썸네일은 화면 정렬이 유지된다.
- 에디터 업로드 이미지 품질 저하 민원이 줄어든다.
- 기존 게시판 스킨의 표시가 크게 깨지지 않는다.
- 썸네일 캐시가 게시판 삭제/수정 시 과도하게 남지 않는다.

## 9. 다음 단계

1. `common/image.php`, `common/board.php`, `board/editor-upload/index.php`의 실제 처리 흐름을 기준으로 설계를 확정한다.
2. 리스트용/본문용 썸네일 정책을 분리하는 design 문서를 작성한다.
3. 스킨별 표시 방식은 default, webzine, gallery, youtube 기준으로 나눠 적용한다.
4. 구현 후 세로 이미지, 대형 이미지, 에디터 본문 이미지, 최신글 노출을 함께 검증한다.
