# 게시판 대댓글 PDCA Plan

## 1. 개요

게시판 상세 화면에서 댓글에 답글을 달 수 있도록 대댓글 기능을 추가한다.  
기존 댓글 구조는 유지하고, 대댓글은 댓글 아래에 들여쓰기 형태로 표시한다.

## 2. 문제 정의

- 현재 댓글은 1단계 평면 구조만 지원한다.
- 댓글에 답글을 달 수 있는 UI가 없다.
- 저장 구조에 `parent_id`는 존재하지만 실제 기능이 연결되어 있지 않다.
- 댓글 스레드가 보이지 않아 긴 토론형 게시판 운영에 한계가 있다.

## 3. 목표

- 댓글에 답글을 달 수 있게 한다.
- 대댓글은 부모 댓글 아래에 시각적으로 구분되게 표시한다.
- 기존 일반 댓글 작성과 관리자 숨김 기능은 그대로 유지한다.
- 댓글 수 집계와 게시글 상세 화면 동작을 깨지 않는다.

## 4. 범위

### 포함

- 댓글 저장 시 부모 댓글 지정
- 댓글 트리 렌더링
- 답글 버튼과 답글 작성 폼
- 일반 댓글과 대댓글의 UI 구분
- 대댓글 권한 및 검증

### 제외

- 무한 중첩형 복잡한 스레드 UI 전면 개편
- 댓글 좋아요, 신고, 멘션 기능
- 실시간 알림
- 댓글 수정/삭제 기능 재설계

## 5. 요구사항

### 5.1 기능 요구사항

| ID | Requirement | Priority | Status |
|----|-------------|----------|--------|
| FR-01 | 사용자는 댓글 아래에 답글을 작성할 수 있어야 한다. | High | Pending |
| FR-02 | 대댓글은 부모 댓글 아래에 들여쓰기되어 보여야 한다. | High | Pending |
| FR-03 | 일반 댓글 작성과 대댓글 작성은 같은 권한 정책을 공유해야 한다. | High | Pending |
| FR-04 | 부모 댓글이 없는 잘못된 대댓글 요청은 거부해야 한다. | High | Pending |

### 5.2 비기능 요구사항

| Category | Criteria | Measurement Method |
|----------|----------|-------------------|
| Performance | 댓글 목록 표시 시 체감 지연이 없어야 함 | 기존 댓글 조회 성능 유지 |
| Security | CSRF 및 권한 검증 유지 | 서버 검증 확인 |
| UX | 답글 폼은 Bootstrap 네이티브 컴포넌트만 사용 | 코드 리뷰 |

## 6. 성공 기준

- 댓글에 답글을 작성할 수 있다.
- 답글은 부모 댓글 바로 아래에 표시된다.
- 잘못된 parent_id는 저장되지 않는다.
- 기존 댓글 숨김 기능이 정상 동작한다.

## 7. 리스크

- 너무 깊은 중첩은 가독성을 해칠 수 있다.
- 답글 폼이 많아지면 상세 화면이 길어질 수 있다.
- 댓글 트리 정렬이 잘못되면 순서가 뒤섞일 수 있다.

## 8. 다음 단계

1. 디자인 문서를 작성한다.
2. 댓글 조회 및 저장 로직을 수정한다.
3. 상세 화면에 답글 UI를 추가한다.
4. 동작 확인 후 보고서로 정리한다.
