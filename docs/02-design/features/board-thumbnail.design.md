# 게시판 썸네일 재설계 Design

## 1. 설계 결론

썸네일 문제의 핵심은 `썸네일 생성`과 `썸네일 표시`가 한 덩어리로 묶여 있다는 점이다.
따라서 저장 정책과 출력 정책을 분리한다.

### 권장안

- **리스트/카드/갤러리**: 썸네일 사용
- **본문 이미지**: 원본 비율 유지
- **에디터 업로드**: 원본 보존
- **최신글/미리보기**: 리스트용보다 작은 별도 썸네일 또는 원본 축소본 사용

## 2. 유창화 썸네일 분석 결과

유창화 스타일의 GD 기반 썸네일 코드는 본질적으로 다음 역할로 나뉜다.

- 이미지 리소스 로드
- 리사이즈 계산
- 저장
- 캐시 경로 생성

이 구조 자체는 단순하고 재사용하기 좋다.
문제는 코드 자체보다도, 현재 프로젝트가 이 리사이즈 결과를 어디에 어떻게 쓰는지에 있다.

즉, 다음 두 지점이 분리되어야 한다.

1. 저장 시점의 리사이즈
2. 화면 표시 시점의 크롭/비크롭

## 3. 출력 정책

### 3.1 리스트

- 카드형, 갤러리형, 웹진형 리스트는 썸네일을 사용한다.
- 리스트용 미디어 박스는 기본적으로 `cover` 성격을 갖는다.
- 다만 세로형 이미지가 자주 중요한 게시판은 `cover`를 강제하지 않고 `contain` 또는 여백 허용으로 전환할 수 있어야 한다.
- 즉, 리스트용 썸네일은 "하나의 방식"이 아니라 "스킨별 허용 정책"으로 본다.

### 3.2 본문

- 본문에 출력되는 이미지는 자르지 않는다.
- 이미지는 `img-fluid` 수준으로 폭만 맞추고 높이는 자동으로 따라가게 한다.
- 이미지가 많아도 사용자가 본문 전체를 자연스럽게 볼 수 있어야 한다.

### 3.3 에디터 업로드

- 업로드된 이미지는 원본 파일을 유지한다.
- 저장 시점 강제 축소는 기본적으로 하지 않는다.
- 필요할 때만 별도 미리보기 썸네일을 생성한다.

### 3.4 최신글/미리보기

- 최신글, 홈 위젯, 짧은 미리보기는 본문과 분리한다.
- 작은 박스에 이미지를 넣을 때는 `contain` 우선으로 설계한다.
- 카드형 홍보 영역처럼 화면 균형이 더 중요하면 `cover`를 제한적으로 사용한다.
- 이 영역은 리스트보다 작은 별도 썸네일 크기를 사용할 수 있다.

## 4. 썸네일 캐시 전략

- 리스트용 썸네일 캐시는 유지한다.
- 본문용은 캐시보다 원본 우선이 적합하다.
- 최신글 위젯처럼 반복 노출이 많은 영역은 별도 작은 썸네일을 둘 수 있다.
- 삭제/수정 시 관련 캐시는 함께 지운다.

## 5. 구현 방향

### A. 공통 리사이즈 함수는 유지

- `common/image.php`의 GD 리사이즈 계층은 유지한다.
- 다만 크롭이 필요한지 여부를 함수 인자로 분리하는 방향을 검토한다.

### B. 화면별 사용처 분리

- `skins/board/default/list.php`
- `skins/board/default/view.php`
- `skins/board/youtube/list.php`
- `skins/board/youtube/view.php`

각 화면에서 같은 썸네일 함수를 쓰더라도, 래퍼 클래스와 표시 방식은 다르게 둔다.

### C. 본문과 리스트의 CSS 역할 분리

- 리스트는 카드 균형 우선
- 본문은 가독성 우선

## 6. 권장 기술 선택

### 썸네일 생성

- GD 기반 현재 구조 유지
- 캐시는 파일 경로 해시 기반 유지
- 원본이 작으면 재생성하지 않고 복사 유지

### 본문 표시

- 크롭 금지
- 원본 비율 유지
- 반응형 너비 적용

### 에디터 업로드

- 원본 보존
- 업로드 시 품질 저하 유발 리사이즈 제거 검토

## 7. 리스크

- 원본 보존으로 인해 저장 용량이 증가할 수 있다.
- 본문 이미지가 큰 경우 페이지 로딩이 무거워질 수 있다.
- 일부 스킨은 기존 `cover` 정렬에 의존하므로, 리스트 표현이 조금 달라질 수 있다.

## 8. 검증 기준

- 세로로 긴 이미지가 본문에서 잘리지 않는다.
- 리스트 카드에서 이미지 비율이 무너지지 않는다.
- 에디터 업로드 이미지의 화질이 유지된다.
- 썸네일 캐시 삭제가 동작한다.
- 유튜브/웹진/갤러리 스킨에서 깨짐이 없다.
