# 게시판 대댓글 Design

## 1. 목표

게시판 상세 화면에서 댓글에 답글을 달 수 있게 한다.  
대댓글은 부모 댓글 하단에 들여쓰기된 카드 형태로 표시하고, 기존 일반 댓글/관리자 숨김 기능은 유지한다.

## 2. 핵심 설계 원칙

- 기존 댓글 테이블의 `parent_id`를 활용한다.
- 댓글 저장은 `post_id`와 `parent_id`를 함께 검증한다.
- 렌더링은 평면 리스트가 아니라 부모-자식 트리로 구성한다.
- UI는 Bootstrap 기본 컴포넌트와 유틸리티만 사용한다.

## 3. 데이터 흐름

1. 사용자가 댓글 또는 답글을 제출한다.
2. 서버는 CSRF, 게시판 권한, 본문 입력값을 검증한다.
3. 답글이면 `parent_id`가 동일 게시글의 댓글인지 확인한다.
4. 저장 후 댓글 카운트를 갱신한다.
5. 상세 화면은 댓글 목록을 트리로 변환해 출력한다.

## 4. 데이터 모델

### 4.1 댓글 엔티티

```php
[
  'id' => int,
  'post_id' => int,
  'parent_id' => ?int,
  'author_id' => ?int,
  'author_name' => string,
  'content' => string,
  'is_hidden' => int,
  'created_at' => string,
  'updated_at' => string,
  'children' => array
]
```

### 4.2 관계

```
[Post] 1 ---- N [Comment]
               |
               +---- 1 ---- N [Reply]
```

실제 구현은 동일 테이블에서 `parent_id`로 계층을 표현한다.

## 5. API / 서버 동작

### 5.1 댓글 작성

| Action | Input | Result |
|--------|-------|--------|
| comment_create | content | 일반 댓글 저장 |
| comment_create | content + parent_id | 대댓글 저장 |

### 5.2 검증

- `content`는 필수다.
- `parent_id`가 있으면 현재 글의 댓글인지 확인한다.
- 권한이 없으면 저장하지 않는다.

## 6. UI 설계

### 6.1 상세 댓글 영역

- 각 댓글은 Bootstrap card 또는 light background block으로 표시한다.
- 대댓글은 부모 댓글보다 한 단계 더 안쪽으로 들여쓰기한다.
- 댓글 상단에는 작성자와 시간, 하단에는 본문을 둔다.
- 댓글마다 `답글` 버튼을 제공한다.
- 답글 버튼은 collapse로 답글 폼을 열도록 한다.

### 6.2 댓글 작성 폼

- 페이지 하단에는 기본 댓글 작성 폼을 유지한다.
- 각 댓글의 답글 폼은 해당 댓글 아래에 접힌 상태로 둔다.
- 답글 폼은 `parent_id` hidden input을 가진다.

## 7. 오류 처리

| Code | Message | Handling |
|------|---------|----------|
| 400 | 잘못된 답글 대상입니다. | 저장 중단 |
| 403 | 댓글 작성 권한이 없습니다. | 경고 표시 |
| 419 | CSRF 검증 실패 | 기존 보안 처리 |

## 8. 테스트 계획

| Type | Target | Method |
|------|--------|--------|
| Manual | 일반 댓글 작성 | 상세 화면에서 등록 확인 |
| Manual | 답글 작성 | 부모 댓글 아래 표시 확인 |
| Manual | 잘못된 parent_id | 저장 거부 확인 |
| Manual | 관리자 숨김 | 기존 기능 유지 확인 |

## 9. 구현 순서

1. 댓글 조회를 트리 구조로 바꾼다.
2. 댓글 저장에 `parent_id`를 연결한다.
3. 댓글 상세 화면에 답글 UI를 추가한다.
4. 유튜브 상세 화면도 동일 구조로 반영한다.
