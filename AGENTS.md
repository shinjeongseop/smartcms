# AGENTS.md instructions for D:\smartcms

## 기본 지침

- 모든 답변은 한국어로 한다.
- 사용자가 질문만 한 경우에는 답변만 하고, 코드 수정은 명시적으로 요청받았을 때만 진행합니다.
- 로컬 변경사항은 커밋 후 GitHub에 푸시한다.
- GitHub Actions 배포가 기본이며, 서버 직접 수정이나 FTP 직접 업로드는 하지 않는다.
- 같은 기능이 여러 화면에 반복되면 공통 모듈을 먼저 봅니다.
- HTML은 구조만 담당하고 이벤트와 로직은 자바스크립트로 분리합니다.
- PHP 파일 안에 `<script>`를 새로 넣는 일은 가급적 피합니다.
- 없는 DOM 요소에 `addEventListener`, `.value`, `.disabled`를 바로 걸지 않고 null 방어를 둡니다.
- `btn-outline-*` 아웃라인 버튼은 사용하지 않습니다.
- 인코딩이 깨진 파일은 수정 전에 UTF-8로 정리합니다.
- SmartGrid 설정은 화면별로 중복하지 말고 공통화 가능성을 먼저 검토합니다.

## 디자인 지침

- 새 페이지, 스킨, 관리자 화면, 컴포넌트는 반드시 루트의 `DESIGN.md`를 먼저 확인하고 따른다.

## 금지사항

- 인라인 스타일 금지.
- 중복 CSS/JS 작성 금지.

## 필수사항

- 공통 CSS는 `common/css/common.css`에 둔다.
- 페이지 전용 CSS가 필요하면 파일로 분리하되 `DESIGN.md` 토큰을 따른다.
- 모듈화, 재사용성, 유지보수성을 우선한다.
